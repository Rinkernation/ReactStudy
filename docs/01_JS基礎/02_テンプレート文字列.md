# JS基礎: テンプレート文字列

JavaScriptにおいて、文字列の中に変数や式を直感的に埋め込むための構文が **テンプレート文字列（Template Literals）** である。
React開発でも、画面に表示するテキストを動的に組み立てたり、クラス名を結合したりする際などに非常に頻繁に使用される。

## 1. 基本的な使い方

テンプレート文字列を使用するには、文字列を囲む記号として、従来のシングルクォート（`'`）やダブルクォート（`"`）の代わりに、**バッククォート（`` ` ``）**を使用する。

バッククォートで囲んだ文字列の中では、`${変数名}` のように記述することで、その部分に変数の値やJavaScriptの式をそのまま展開（埋め込み）することができる。

### 比較コード例

```javascript
const name = "テストユーザー";
const age = 20;

// 【従来の書き方】 +演算子で結合
// 文字列と変数を交互に + で繋ぐ必要があり、読みにくくミスが起きやすい
const message1 = "私の名前は" + name + "です。年齢は" + age + "です。";

// 【推奨される書き方】 テンプレート文字列
// 全体をバッククォートで囲み、変数を埋め込みたい部分に ${} を使うだけで済む
const message2 = `私の名前は${name}です。年齢は${age}です。`;
```

## 2. テンプレート文字列の利点

1. **可読性の向上**
   `+` 演算子とクォーテーションを大量に並べる必要がなくなり、最終的にどのような文章になるのかが一目で分かるようになる。
2. **改行がそのまま反映される**
   従来の文字列では、改行を表現するために `\n` を挿入する必要があった。一方、テンプレート文字列ではエディタ上で改行（Enter）したものが、そのまま文字列の改行として認識される。

```javascript
// 複数行の文字列も、見た目のまま直感的に記述できる
const multiLineText = `
これは1行目です。
これは2行目です。
`;
```

## 3. Reactにおける活用シーン

React（JSX）においては、JavaScriptの変数を組み合わせてクラス名（CSS）を指定する際などで特に重宝する。

```javascript
// （例）isErrorがtrueなら 'error' というクラス名を足す
const className = `message-box ${isError ? 'error' : ''}`;
```
