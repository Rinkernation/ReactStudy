# JS基礎: 変数宣言（var, let, const）

JavaScriptにおける変数宣言の仕組みと、それぞれの特徴について解説する。
モダンなフロントエンド開発（Reactなど）においては、予期せぬバグを防ぐために `var` の使用は非推奨とされている。**基本的には `const` を使用し、値の変更（再代入）がどうしても必要な場合のみ `let` を使用する**のがベストプラクティスである。

## 1. var の特徴
- **再代入**: 可能（上書き可能）
- **再宣言**: 可能
- **スコープ**: 関数スコープ

同じ変数名を誤って複数回宣言（再宣言）できてしまうため、コードが長くなった際に意図せず他の変数を上書きしてしまうリスクが非常に高い。そのため、現在の開発では**非推奨**である。

```javascript
var val1 = "var変数";
val1 = "var変数を上書き"; // 再代入可能
var val1 = "var変数を再宣言"; // 再宣言も可能（バグの温床になる）
```

## 2. let の特徴
- **再代入**: 可能（上書き可能）
- **再宣言**: 不可
- **スコープ**: ブロックスコープ

再宣言が不可能なため、`var` よりも安全である。ループのカウンター変数など、値の変更（上書き）が前提となる場合に使用する。

```javascript
let val2 = "let変数";
val2 = "let変数を上書き"; // 再代入は可能
// let val2 = "let変数を再宣言"; // エラーになる（安全）
```

## 3. const の特徴
- **再代入**: 不可（上書き不可）
- **再宣言**: 不可
- **スコープ**: ブロックスコープ

再代入も再宣言もできないため、コードを後から読んだときに「この値は絶対に変わらない」ということを保証できる。最も安全な変数宣言である。

```javascript
const val3 = "const変数";
// val3 = "const変数を上書き"; // エラーになる
// const val3 = "const変数を再宣言"; // エラーになる
```

## 4. const におけるオブジェクトと配列の変更
`const` は「再代入」を禁止するものであるが、**オブジェクトや配列の中身（プロパティや要素）の変更は可能**である。変数が指し示す「参照先（箱そのもの）」を別のものにすり替えることはできないが、箱の中身を書き換えることは許可されている。

### オブジェクトの例
```javascript
const obj = {
  name: "テスト",
  age: 20
};

// プロパティの変更や追加は可能（エラーにならない）
obj.name = "変更されたテスト";
obj.address = "Tokyo";
console.log(obj); // { name: "変更されたテスト", age: 20, address: "Tokyo" }

// オブジェクトそのものを再代入しようとするとエラーになる
// obj = { name: "別のオブジェクト" }; // TypeError
```

### 配列の例
```javascript
const arr = ["dog", "cat"];

// 要素の変更や追加は可能（エラーにならない）
arr[0] = "bird";
arr.push("fish");
console.log(arr); // ["bird", "cat", "fish"]

// 配列そのものを再代入しようとするとエラーになる
// arr = ["apple"]; // TypeError
```
