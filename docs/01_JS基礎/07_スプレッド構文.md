# JS基礎: スプレッド構文

配列やオブジェクトの中身を「展開」したり、逆に「まとめ」たりするための便利な構文が **スプレッド構文（`...`）** である。
Reactでは、状態（State）を更新する際や、コンポーネントにPropsをまとめて渡す際に必須となるテクニックである。

## 1. 配列の展開

配列の前に `...` をつけることで、配列の中身を一つずつ展開して取り出すことができる。

```javascript
const arr1 = [1, 2];

// 【通常の書き方】
console.log(arr1); // [1, 2] （配列として出力される）

// 【スプレッド構文を用いた書き方】
console.log(...arr1); // 1 2 （中身が展開されて出力される）

// 関数の引数に渡す際にも便利である
const sumFunc = (num1, num2) => console.log(num1 + num2);

// 【通常の書き方】 インデックスを指定
sumFunc(arr1[0], arr1[1]); 

// 【スプレッド構文を用いた書き方】
sumFunc(...arr1); 
```

## 2. 要素をまとめる（残余引数）

分割代入と組み合わせることで、一部の要素を取り出しつつ、残りの要素を新しい配列として「まとめる」ことができる。

```javascript
const arr2 = [1, 2, 3, 4, 5];

const [num1, num2, ...arr3] = arr2;

console.log(num1); // 1
console.log(num2); // 2
console.log(arr3); // [3, 4, 5] （残りの要素が配列としてまとまる）
```

## 3. 配列のコピーと結合

React開発において最もよく使われるのが、配列（またはオブジェクト）の「コピー」と「結合」である。

```javascript
const arr4 = [10, 20];
const arr5 = [30, 40];

// 【Reactで推奨される書き方】 コピー
const arr6 = [...arr4];
console.log(arr6); // [10, 20]

// 【Reactで推奨される書き方】 結合
const arr7 = [...arr4, ...arr5];
console.log(arr7); // [10, 20, 30, 40]
```

### 🚨 参照渡しに注意（スプレッド構文を使う理由）
JavaScriptでは、配列を `=` でそのまま別の変数に代入すると、中身ではなく「参照元（メモリ上の場所）」がコピーされてしまう。
そのため、コピー先を変更するとコピー元まで書き換わってしまうというバグ（意図しない副作用）が発生しやすい。

```javascript
const arr4 = [10, 20];

// そのまま代入すると「参照（データの場所）」がコピーされる
const arr8 = arr4;
arr8[0] = 99;

console.log(arr4); // [99, 20] （コピー元まで書き換わってしまう）
```

スプレッド構文を使って展開してから新しい配列 `[]` に入れることで、完全に別の配列として安全にコピー（ディープコピー）を作成できる。
