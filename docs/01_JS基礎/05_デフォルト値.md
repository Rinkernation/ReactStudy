# JS基礎: デフォルト値

変数に値が渡ってこなかった場合（`undefined` の場合）に備えて、初期値を設定できるのが **デフォルト値（Default Value）** の機能である。主に関数の引数や、分割代入の際に使用される。

## 1. 関数の引数におけるデフォルト値

関数呼び出し時に引数が省略された場合に、エラーや意図しない表示（画面に `undefined` と表示されてしまうなど）を防ぐために設定する。

```javascript
// nameが渡されなかった場合は、自動的に "ゲスト" が代入される
const sayHello = (name = "ゲスト") => {
  console.log(`こんにちは！${name}さん！`);
};

sayHello("テストユーザー"); // 「こんにちは！テストユーザーさん！」
sayHello(); // 引数なし → 「こんにちは！ゲストさん！」
```

## 2. 分割代入におけるデフォルト値

APIからのデータ取得時など、「特定のプロパティが存在しないかもしれない」というケースで非常に役立つ安全対策である。

```javascript
const userProfile = {
  age: 30
  // name プロパティが存在しないオブジェクト
};

// nameが存在しない場合は "ゲスト" をデフォルト値として設定する
const { age, name = "ゲスト" } = userProfile;

console.log(age);  // 30
console.log(name); // "ゲスト" （エラーにならず安全に処理される）
```

## 3. Reactでの活用シーン
Reactでは、親から渡されるデータ（Props）が設定されていない場合の「初期表示（何もデータがない時の安全な表示）」を担保するために、分割代入と組み合わせてデフォルト値が頻繁に使用される。
