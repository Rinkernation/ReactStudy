# SPA（Single Page Application）の仕組みと利点

Reactは、**SPA（Single Page Application）** と呼ばれるWebアプリケーションを構築するためのライブラリである。SPAの仕組みと、従来型のWebサイトと比較した際の利点を解説する。

## 1. 従来型のWebサイトとSPAの違い

### 従来型のWebシステム（マルチページアプリケーション）
ユーザーがリンクをクリックするたびに、サーバーに新しいHTMLファイルを要求し、画面全体を読み込み直す方式である。
- `http://example.com/` にアクセス → サーバーが `top.html` を作成して返す
- `http://example.com/profile.html` にアクセス → サーバーが `profile.html` を作成して返す

### SPA（Single Page Application）
ユーザーが最初にアクセスした時に、枠組みとなる簡素な単一の `index.html` と、必要なJavaScript（Reactなど）を一度だけ読み込む。
その後のページ遷移は、JavaScriptが**必要なデータ（JSONなど）のみをサーバーから取得し、画面の必要な部分だけを書き換える**方式である。

## 2. SPAの利点

SPAを採用することで、ユーザー体験（UX）と開発体験（DX）の両面で大きなメリットが得られる。

### ① ユーザーにとっての利点：表示速度と操作性の向上
従来型のWebでは、ページ遷移のたびに画面全体（ヘッダーやフッターも含め）を再読み込みするため、画面が白くチラつく（リロードが発生する）。
一方、SPAでは初回の読み込み以降は**変更があった部分（メインコンテンツなど）のみを部分的に書き換える**（ここでReactの仮想DOMが活躍する）。
通信量が少なく、画面のチラつきもないため、まるでスマートフォンのネイティブアプリのような高速でサクサクとした操作性を実現できる。

### ② 開発者にとっての利点：コンポーネント分割による保守性の向上
従来型の開発では、複数ページで共通のデザイン（例：ボタン、ヘッダー）を使用する場合、各HTMLファイルに同じコードを記述する必要があった。
SPA（特にReact）では、UIをJavaScriptで構築するため、画面を**「コンポーネント（部品）」**という単位で細かく分割・管理できる。
これにより以下の恩恵を受けられる。
- **再利用性**: 一度作成したコンポーネント（例：`Button` コンポーネント）を、様々な画面で使い回すことができる。
- **保守性**: デザイン変更や機能修正があった際、該当する1つのファイルを修正するだけで、そのコンポーネントを使用している全ての画面に変更が適用される。
