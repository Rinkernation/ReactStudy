# Reactの利点と特徴

## 1. DOMとは？
**DOM (Document Object Model)** とは、HTMLドキュメントをツリー構造のオブジェクトとして表現したものである。
JavaScriptなどのプログラムから、Webページの見た目や構造（HTML要素）を操作するための「窓口（API）」の役割を果たす。

## 2. Reactと仮想DOM (Virtual DOM)
**Reactでは実際のDOMを直接編集するのではなく、「仮想DOM（Virtual DOM）」を用いる。**

### 仕組み
1. **仮想DOMの作成**: Reactは、実際のDOMのコピーのようなものを「メモリ上」に仮想DOMとして保持する。
2. **比較（Diffing）**: アプリケーションのデータ（状態＝State）が更新されると、Reactは新しい仮想DOMを作成し、直前の仮想DOMとの「差分」を計算する。
3. **反映（Reconciliation）**: 計算結果をもとに、**変更があった部分だけ**を実際のDOMに反映させる。

### なぜ直接DOMを編集しないのか？（利点）

1. **パフォーマンスの向上**
   - 実際のDOMを操作する（画面を再描画する）処理は、ブラウザにとって非常に計算コストが高い処理である。
   - 仮想DOMの作成と差分計算はメモリ上で行われるため非常に高速である。必要最低限の実際のDOMだけを更新することで、パフォーマンスの低下を防ぐ。
2. **コードの保守性と開発体験の向上（宣言的UI）**
   - 従来のDOM操作（例：jQueryなど）では、「ボタンが押されたら、あの要素の色を変えて、この要素のテキストを書き換える」という**手続き型**の処理を記述する必要があった。
   - Reactでは、「現在のデータ（状態）なら、UIはこのように表示される」という**宣言的**なコードを書く。複雑なDOMの操作や更新処理はReactの仮想DOMが裏側で全て行うため、開発者は「UIがどう見えるべきか」に集中でき、バグが少なく保守しやすいコードになる。
