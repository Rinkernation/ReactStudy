# 開発環境の構築手順：Vanilla JS編

本プロジェクトの学習（バニラJS・Reactの基礎概念）を進めるための、ローカル開発環境の構築および起動手順についてまとめる。

## 1. 前提条件（必須ツール）

以下のツールがインストールされている必要がある。

- **Node.js**: JavaScriptの実行環境およびパッケージ管理（npm）に必要。
  - 推奨バージョン: LTS（長期サポート版）
  - 確認方法: ターミナルで `node -v` を実行し、バージョンが表示されればOK。

## 2. ローカルサーバーの起動（npx serve）

HTMLファイルやJavaScriptファイルをブラウザで正しく動作させるため（特にモジュール機能などを利用する場合）、ローカルサーバーを起動する必要がある。
本プロジェクトでは、軽量なサーバーツールである `serve` を使用する。

### 起動手順

1.  ターミナルを開き、`vanilla_js` ディレクトリへ移動する。
2.  以下のコマンドを実行する。

```bash
npx serve
```

- **npx** とは：パッケージをローカルにインストールすることなく、一時的にダウンロードして直接実行するためのコマンドである。
- 初回実行時には `Need to install the following packages: serve. Ok to proceed? (y)` と聞かれる場合がある。その際は `y` を入力して続行する。

### アクセス方法

実行後、ターミナルに以下のようなメッセージが表示される。

```text
┌──────────────────────────────────────────────────┐
│                                                  │
│   Serving!                                       │
│                                                  │
│   - Local:    http://localhost:3000              │
│   - Network:  http://192.168.1.5:3000            │
│                                                  │
└──────────────────────────────────────────────────┘
```

`Local:` に表示されているURL（例: `http://localhost:3000`）をブラウザで開くことで、開発中の画面を確認できる。

## 3. 学習用ファイルの構成と運用

`vanilla_js` ディレクトリ内は以下の構成となっている。

- **index.html**: メインのHTMLファイル。
- **src/index.js**: 教材のサンプルコードを試すためのファイル。
- **src/practice.js**: 演習問題を解くための作業用ファイル。

### JSファイルの切り替え方法

実行するJavaScriptを切り替えるには、`index.html` 内の `<script>` タグの参照先を書き換える。

```html
<!-- 例：演習問題を解く場合 -->
<script src="src/practice.js"></script>
```
